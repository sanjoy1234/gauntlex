# Contributing to GAUNTLEX

Thanks for considering a contribution. The real guidance lives in two places, both linked here so GitHub's own tooling (the "Contribute" prompt on new issues/PRs) finds it automatically:

- **[Deep Dive → Contributing](docs/DEEP_DIVE.md#contributing)** — ground rules and overview.
- **[CONTRIBUTING-PLAYBOOKS.md](CONTRIBUTING-PLAYBOOKS.md)** — step-by-step playbooks for the most common contribution types: adding a policy domain, a language profile, or a feature.

A few starting points if you're not sure where to begin:

- **Questions, feedback, or "I built X with this" show-and-tell:** [GitHub Discussions](https://github.com/sanjoy1234/gauntlex/discussions) — this is the right place, not Issues.
- **Bugs or concrete feature requests:** [Issues](https://github.com/sanjoy1234/gauntlex/issues), including anything labeled `good first issue`.
- **Security vulnerabilities:** see [SECURITY.md](SECURITY.md) — please don't file these as public issues.

All contributions are expected to follow the [Code of Conduct](CODE_OF_CONDUCT.md).
