CWE-285 Missing authorization claims in JWT leading to privilege escalation HIGH ❌ UNMITIGATED

The login endpoint creates a JWT containing only the 'sub' (username) claim, without any role or permission information. If downstream services rely solely on the presence of a valid JWT (signature verification) to grant access to sensitive functions (e.g., admin endpoints) without checking authorization claims, an attacker who obtains a legitimate token for a low‑privilege user can bypass authorization and access privileged resources.

Fix: Review the CWE definition and apply defense-in-depth controls.
