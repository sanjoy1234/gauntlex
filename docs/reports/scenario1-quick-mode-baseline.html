CWE-330 Predictable JWT Secret Key Fallback HIGH ❌ UNMITIGATED

The application uses a hardcoded, predictable fallback JWT secret key ('a-fallback-secret-key-for-demo-only-change-in-production') when the JWT_SECRET_KEY environment variable is not set. An attacker can exploit this by forging valid JWT tokens for any user (e.g., setting user_id to 1 and username to 'admin') using the known secret key, leading to complete authentication bypass and privilege escalation.

CWE-20 Type confusion in username/password fields leading to server error MEDIUM ❌ UNMITIGATED

The login endpoint does not validate that the 'username' and 'password' fields are strings. If a non-string value (e.g., integer, list) is provided for either field, the application will raise an AttributeError when attempting to call string methods (like `.strip()` or `.lower()`), resulting in a 500 Internal Server Error. An attacker can exploit this to cause denial-of-service by sending malformed requests that trigger server errors.

Fix: Review the CWE definition and apply defense-in-depth controls.
CWE-770 Unlimited login attempts for authenticated users due to exemption condition MEDIUM ❌ UNMITIGATED

The `exempt_successful_login` function exempts requests with valid credentials from the rate limit (5/minute). An attacker who has obtained valid credentials can send an unlimited number of login requests without being rate limited. Each request consumes server resources: database connection, CPU for JWT signing and password verification, and log storage. This can lead to denial-of-service via resource exhaustion.

Fix: Review the CWE definition and apply defense-in-depth controls.
