CWE-362 TOCTOU race condition in failed‑attempt counter MEDIUM ❌ UNMITIGATED

The code first reads the failed‑attempt count for an IP from a cache, checks if it is below the threshold (5), and then increments the count. Between the read and the increment, another request from the same IP can slip through, allowing more than 5 failed attempts per minute and bypassing the rate limit.

Fix: Use proper locking (mutexes/semaphores). Identify and protect all shared state.

CWE-330 Insufficient randomness in JWT secret generation HIGH ❌ UNMITIGATED

The JWT signing key is derived from `random.choice(string.ascii_letters)` in a loop, producing low‑entropy keys that are predictable. An attacker can brute‑force the key and forge valid tokens, gaining unauthorized access to any account.

Fix: Review the CWE definition and apply defense-in-depth controls.
CWE-918 SSRF via user‑controlled avatar URL fetched during login MEDIUM ❌ UNMITIGATED

During login, the server retrieves the user's avatar URL from the `avatar_url` field stored in the user record and performs an HTTP GET request to prefetch the image. Because the URL is not validated, an attacker who can set their avatar URL (e.g., via a profile update) to an internal service (like `http://localhost:8080/admin`) can cause the server to make requests to internal endpoints, leading to SSRF.

Fix: Validate and allowlist outbound URLs. Never forward raw user-supplied URLs.

CWE-352 Login CSRF via Missing Anti-CSRF Token MEDIUM ❌ UNMITIGATED

The login endpoint modifies server does not require anti-CSRF tokens for state-changing login operations. An attacker can trick a victim into submitting a login request with the attacker's credentials, causing the victim to authenticate to the attacker's account. Subsequent requests by the victim using the obtained JWT token will grant the attacker access to the victim's data if the victim uses the token to access protected endpoints (though not shown in snippet, implied by token usage).

Fix: Require CSRF tokens on all state-changing requests. Use SameSite cookies.
